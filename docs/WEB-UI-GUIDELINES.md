# Web UI Guidelines

Практические рекомендации по построению web UI, выведенные из проекта 1230UI. Документ сфокусирован на frontend: технологический стэк, принципы интернационализации, дизайн-токены, доступность, мобильная адаптация и хранение визуальных активов. Конкретные значения цветов не приводятся — только принципы и токены.

## 1. Технологический стэк

| Категория | Технология | Зачем |
|-----------|-----------|-------|
| Сборка / dev-сервер | Vite | Быстрый HMR, esbuild-компиляция, нативная поддержка TypeScript и CSS-импортов |
| UI-фреймворк | React 19 + TypeScript (strict) | Компонентная модель, type-safety, экосистема хуков |
| Маршрутизация | react-router-dom (BrowserRouter) | Декларативные маршруты, поддержка вложенных layout'ов, code-splitting через `lazy()` |
| Стилизация | Tailwind CSS v4 + `@tailwindcss/vite` | Утилитарные классы, токенизация через `@theme`, zero-runtime |
| State management | Zustand (+ `persist` middleware) | Минимальный boilerplate, поддержка персистентности в localStorage из коробки |
| Иконки | lucide-react | Tree-shakeable SVG-иконки, единый стиль |
| Серверный state | Custom hooks + polling (`useEffect` + `setInterval`) | Полный контроль над частотой опроса, очисткой таймеров |
| Виртуализация списков | react-virtuoso | Эффективный рендеринг длинных списков (сессии, сообщения) |
| Markdown | react-markdown + remark-gfm + rehype-highlight + highlight.js | Рендеринг форматированного текста с подсветкой кода |
| Локализация | i18next + react-i18next + i18next-browser-languagedetector | i18n с автоопределением языка, fallback'ом, pluralSeparator для русского |
| Санитизация | xss, zod | XSS-защита для пользовательского контента, валидация API-ответов |
| Тестирование | Vitest + Testing Library | Единый раннер с Vite, быстрые unit/integration тесты |
| Линтинг | ESLint + typescript-eslint + react-hooks plugin | Контроль качества кода, защита от типичных ошибок React |
| HTTP | Custom `api.ts` (fetch-обёртка) | Типизированные вызовы, единая обработка ошибок |

### Почему этот стэк

- **Минимум абстракций** — Zustand вместо Redux, fetch вместо axios. Меньше кода, меньше поверхность для багов.
- **Tailwind v4 с `@theme`** — дизайн-токены живут в одном месте, нет разрыва между "токенами" и "классами".
- **TypeScript strict** — большинство ошибок ловится на этапе компиляции, а не в продакшне.
- **Vite proxy** — `/api/*` проксируется на backend в dev-режиме, в продакшне — через тот же префикс за reverse-proxy.

## 2. Структура проекта

```
src/
├── main.tsx              # Точка входа, импорт CSS + i18n
├── App.tsx               # Router + Provider'ы (ErrorBoundary, Toast)
├── index.css             # Глобальные стили, @theme, @font-face
├── components/           # Переиспользуемые UI-компоненты
│   ├── Layout.tsx        # Корневой layout (Navbar + Outlet + footer)
│   ├── Navbar.tsx
│   ├── MobileNav.tsx     # Мобильная навигация (fixed bottom)
│   ├── Modal.tsx         # Модалки с focus-trap
│   ├── Toast.tsx         # Provider + viewport
│   ├── ErrorBoundary.tsx
│   └── settings/         # Специфичные для настроек компоненты
├── pages/                # Route-уровневые компоненты (lazy)
├── hooks/                # Кастомные хуки (бизнес-логика)
├── store/                # Zustand-сторы (один стор = одна фича)
├── lib/                  # Утилиты, API-клиент, цветовые палитры
├── i18n/                 # i18next конфиг + локали
│   └── locales/
│       ├── en/translation.json
│       ├── ru/translation.json
│       ├── de/translation.json
│       └── es/translation.json
├── styles/               # Дополнительные CSS-файлы
├── types/                # Глобальные TypeScript-типы
├── assets/               # Статические медиа-файлы (импортируются в код)
└── applications/         # Реестр встраиваемых "приложений"
    ├── registry.ts       # Маппинг id → компонент
    └── types.ts          # Контракт ApplicationComponentProps
```

### Принципы организации

- **Один стор — одна фича.** Не делать god-store. Каждый `useXxxStore` отвечает за свой срез состояния.
- **Хуки отделяют логику от UI.** Компонент описывает только разметку и связки, вся императивная логика — в `hooks/`.
- **Lazy-загрузка страниц.** Каждая страница — отдельный чанк, который грузится при переходе.
- **Переиспользуемые компоненты** лежат в `components/`, страничные — в `pages/`. Никогда наоборот.

## 3. Интернационализация (i18n)

### Конфигурация

- **Библиотека:** `i18next` + `react-i18next` + `i18next-browser-languagedetector`.
- **Fallback:** английский (`fallbackLng: 'en'`).
- **Хранилище выбора:** `localStorage` под ключом `i18nextLng`.
- **Порядок определения:** `['localStorage', 'navigator']` — пользовательский выбор приоритетнее системного.
- **Pluralization:** для русского используется `pluralSeparator: '_'` и формы `_one` / `_few` / `_many` / `_other`.
- **Интерполяция:** `{{count}}`, `{{name}}` и т.д. `escapeValue: false` (React сам экранирует).

### Структура переводов

Имена ключей строятся по дереву фич:

```
common.cancel
common.save
nav.home
nav.sessions
settings.providers.title
settings.providers.addProvider
chat.send
chat.stop
```

### Правила

1. **Все строки UI — через `t(...)`.** Никаких хардкодов в JSX.
2. **Единая вложенность JSON.** Лучше `settings.providers.title`, чем плоский `settingsProvidersTitle`.
3. **Plural-формы обязательны** для счётчиков (`{{count}} session` / `{{count}} sessions`).
4. **Брендовые и версионные строки** тоже переводимы (даже если бренд один — это упрощает локализацию).
5. **Fallback работает по ключам** — если в `ru` нет ключа, подтянется `en`. Не дублируйте структуры вручную.
6. **Добавление языка = добавление файла** в `locales/<lng>/translation.json` + регистрация в `i18n/index.ts` (`resources` и `supportedLngs`).

## 4. Дизайн-система и токены

### Принцип единого источника правды

Все цвета, радиусы и тени объявляются **один раз** в `@theme` внутри `index.css` и используются как Tailwind-классы (`bg-bg-primary`, `text-fg-secondary`, `rounded-md`). Это исключает "магические хексы" в компонентах.

### Категории токенов

| Группа | Назначение | Примеры имён |
|--------|-----------|--------------|
| `bg-*` | Фоны всех уровней | `bg-primary` (основной), `bg-secondary` (фон страницы), `bg-elevated` (карточки/модалки), `bg-muted` (неактивные зоны), `bg-overlay` (затемнение модалок) |
| `fg-*` | Цвета текста по иерархии | `fg-primary` (заголовки), `fg-secondary` (основной текст), `fg-muted` (подписи/мета), `fg-inverse` (текст на цветном фоне) |
| `border-*` | Границы | `border-default` (обычные), `border-strong` (выделенные) |
| `accent` + варианты | Главный интерактивный цвет | `accent` (фон кнопок), `accent-hover`, `accent-soft` (плашки/бэйджи), `accent-fg` (текст на soft-фоне) |
| `success` / `warning` / `danger` | Семантические статусы | Каждый с `*` и `*-soft` (приглушённый фон для бэйджей) |
| Радиусы | `radius-sm` / `md` / `lg` / `xl` | Маленькие/средние/большие/hero |
| Тени | `shadow-sm` / `md` / `lg` / `xl` | Четыре уровня глубины |

### Правила использования цветов

- **Минимум цветов в проекте.** Реально используется 5 групп: bg, fg, border, accent, status (success/warning/danger). Любой новый цвет должен быть оправдан.
- **Status-цвета** — только для алертов, тостов, ошибок валидации, индикаторов состояния. **Никогда** для оформления кнопок или иконок без семантики.
- **Soft-варианты** — для фона (плашка/banner), основной цвет — для бордера/иконки/текста.
- **Hover-варианты** — у основного цвета (bg) всегда есть `*-hover` вариант.
- **Иерархия текста** — три уровня: primary (заголовки), secondary (основной текст), muted (метаданные/подсказки). Не больше.

### Темы (light / dark)

- **Класс `.dark`** на `<html>` управляет темой. В Tailwind это объявляется через `@custom-variant dark (&:where(.dark, .dark *))`.
- **В dark-варианте** переопределяются только значения тех же токенов — не вводятся новые.
- **`isDarkMode`** хранится в Zustand-сторе с `persist` в localStorage.
- **Default — dark** (или любой, который нравится пользователю), но переключатель должен быть в один клик.
- **Системная тема** (`prefers-color-scheme`) — не единственный источник; пользовательский выбор приоритетнее.
- **Print-стили** принудительно переключают на light, скрывают навигацию и формы ввода.

### Радиусы и тени

- **Скейл радиусов фиксированный:** 4 уровня (sm/md/lg/xl). Не плодить промежуточные.
- **Тени — из 4 уровней** по принципу "чем выше элемент, тем больше тень". Модалки и floating-панели используют самые глубокие.
- **Для fixed-панелей** (mobile nav, chat input) — направленная тень (например, `shadow-[0_-2px_8px_rgba(0,0,0,0.06)]`) снизу, чтобы визуально отделить от контента.

### Фокус и доступность

- **Глобальный outline** для `:focus-visible` (2px solid, offset 2px) — обеспечивает видимый фокус на клавиатурной навигации.
- **На input/select/textarea** outline сбрасывается, фокус показывается через `box-shadow` (ring) — чтобы не "прыгал" layout.
- **Цвет фокус-кольца** совпадает с `accent` (но чуть светлее для контраста с белым фоном).
- **`prefers-reduced-motion: reduce`** глобально отключает анимации и плавные переходы (`@media (prefers-reduced-motion: reduce)`).

## 5. Типографика

- **Системный sans-serif стек** для UI: `-apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', sans-serif`. Не грузить Inter/Roboto с CDN — это лишний вес.
- **Моноширинный стек** — `JetBrains Mono` (через Google Fonts) + fallback на системный моно. Используется для: кода, числовых значений, бренда, бэйджей.
- **Кастомный брендовый шрифт** (например, `IBM Plex Mono` для лого) — загружается локально из `public/fonts/` в формате `woff2`, объявляется через `@font-face` с `font-display: swap`.
- **Базовый размер** — `font-size: clamp(14px, 0.5vw + 13px, 16px)` на `<html>`. Это даёт плавное масштабирование от 14px (мобильный) до 16px (десктоп) без media-queries.
- **Сглаживание:** `-webkit-font-smoothing: antialiased`, `-moz-osx-font-smoothing: grayscale`.

## 6. Скроллбары

- **Webkit + Firefox** стилизованы глобально: ширина 8px, цвет — `border-default` (нейтральный), трек прозрачный.
- **Hover** — более тёмный оттенок `fg-muted` (но не яркий — это не CTA).
- **В dark-теме** — более светлые оттенки, чтобы были видны на тёмном фоне.

## 7. Работа с модалками

Каждая модалка — отдельный компонент со встроенной accessibility:

- **`role="dialog"` + `aria-modal="true"`** + `aria-labelledby` на заголовке.
- **Focus-trap:** при открытии фокус уходит на первый focusable-элемент внутри; `Tab`/`Shift+Tab` зацикливается; при закрытии фокус возвращается на элемент-инициатор (`document.activeElement` сохраняется при mount).
- **`Escape` закрывает** (если `closeOnEsc: true`).
- **Клик по бэкдропу закрывает** (если `closeOnBackdrop: true`).
- **Body scroll lock** — `document.body.style.overflow = 'hidden'` на время открытия, восстанавливается в cleanup.
- **Размеры:** `sm` (max-w-sm), `md`, `lg`, `xl` (max-w-2xl). Жёстко фиксированный набор.
- **Внутренний скролл:** `max-h-[80vh] flex flex-col` — заголовок и футер фиксированы, контент скроллится.
- **Каждый экземпляр модалки** использует общий `<Modal>`-компонент, а не пишет свой.

## 8. Тосты (уведомления)

- **Глобальный `ToastProvider`** в корне приложения. Контекст `ToastApi` доступен через хук `useToast()`.
- **Типы:** `success | error | info | warning` — каждый со своей иконкой (`CheckCircle2`, `AlertCircle`, `Info`, `AlertTriangle`) и цветовой схемой (bg/border/text + иконка).
- **Длительность по умолчанию:** 4000 мс. Можно переопределить на конкретный вызов.
- **Расположение:** fixed top-right, `z-50`, `pointer-events-none` на контейнере и `pointer-events-auto` на самих тостах (чтобы клики проходили сквозь пустоту между ними).
- **Accessibility:** `role="status"` + `aria-live="polite"` — скринридеры прочитают, но не перебьют текущую речь.
- **Кнопка закрытия** есть у каждого тоста (`aria-label` берётся из i18n).
- **Auto-dismiss через `setTimeout`** хранится в `Map<id, timerId>` и очищается при unmount/dismiss.

## 9. Error Boundary

- **Классовый компонент** на самом верху дерева (`<ErrorBoundary>` оборачивает всё в `App.tsx`).
- **При ошибке** — фолбэк-экран с: иконкой, заголовком (i18n), сообщением ошибки, раскрывающимся `<details>` со стектрейсом, кнопкой "Перезагрузить страницу".
- **Кнопка сброса** — `window.location.reload()` (полный сброс состояния).
- **Логирование** — `console.error` с error и errorInfo (на проде сюда можно подключить Sentry/аналог).

## 10. Хуки (паттерны)

### Полинг статусов

- Каждый сервисный поллер — отдельный хук (`useHermesStatusPoll`, `useOpenCodeStatusPoll`).
- **Запускается в Layout**, чтобы работал на всех страницах.
- **Очистка таймера** в `useEffect` cleanup — обязательна.
- **Интервал** подбирается исходя из частоты обновления бэкенда (обычно 2–5 секунд).

### Клавиатурные сокращения

- Хук `useKeyboardShortcuts(arrayOfShortcuts)`.
- Поддержка `ctrlKey` / `metaKey` / `shiftKey` (на Mac и Windows одна комбинация работает везде).
- `preventDefault: false` для тех шорткатов, которые не должны ломать браузерное поведение.
- Ref-паттерн для callback'ов, чтобы не пересоздавать listener при изменении deps.

### Swipe-жесты

- Хук `useSwipe<T>` возвращает `{ ref, translateX, swiping, reset }`.
- **Threshold** (по умолчанию 80px) и **maxTranslate** (160px) — настраиваемые.
- **Long-press** (500 мс по умолчанию) — отдельный колбэк.
- **`data-swipe-ignore`** — атрибут для исключения зон из жестов (например, текстовое поле, в котором идёт выделение).
- **Горизонтальная блокировка** — если движение в основном вертикальное (скролл), горизонтальный жест не активируется.
- **Все обработчики passive: true** где возможно — не блокирует скролл.

### Мобильное определение

- Хук `useMobile()` использует media query `(max-width: 1023px), (pointer: coarse) and (hover: none)`.
- **Планшеты** с тачскрином без мыши считаются мобильными — это правильный UX (у них нет hover).
- **Слушатель `change`** на `matchMedia` — ресайз без перезагрузки.

### Цвет по времени суток (опционально)

- Хук `useTimeBracketColor()` — палитра ассоциирует час с "настроением" (полночь/рассвет/утро/полдень/закат/ночь).
- Обновление каждую минуту через `setInterval`.
- Применяется к декоративным элементам (логотип, акцентные плашки), не к функциональным кнопкам.

## 11. Кастомные палитры для пользовательских сущностей

Если пользователь может создавать сущности, которым нужен уникальный цвет (ассистенты, проекты, теги и т.д.):

- **Зафиксировать ограниченный набор** (8 цветов: blue, green, purple, red, orange, yellow, pink, gray).
- **Для каждого цвета** хранить пакет Tailwind-классов: `bg-*`, `bgSubtle` (приглушённый фон), `text-*`, `border-*`, `ring-*`, `accent` (иконка/ссылка).
- **Обязательно** light + dark вариант (через `dark:`).
- **Fallback** — `gray` (нейтральный), если значение null/undefined/неизвестно.
- **Helper-функция** `getColorClasses(color)` всегда возвращает валидный объект — никаких null в UI.

## 12. Расширяемая архитектура (applications)

Если продукт предполагает встраиваемые "приложения" (файловый менеджер, превью, чат с облаком и т.д.):

- **Контракт** `ApplicationComponentProps`: `{ sessionId, config }` — стандартный интерфейс.
- **Registry:** `Record<id, ComponentType>` — id из backend, компонент из фронта.
- **Точка встраивания** — боковая панель, модалка, или полноэкранный режим.
- **Плейсхолдер** для нереализованных приложений — показывает, что фича существует, но не готова.

## 13. Workspace и распределение ответственности

Начиная с v0.9.3 основная поверхность чата — это **Workspace**: единый shell на маршруте `/sessions` с 3-табовым хедером (Sessions / Hermes / OpenCode). См. [WORKSPACE.md](WORKSPACE.md) для архитектурных деталей.

### Принципы Workspace

- **Единая оболочка вместо отдельных страниц.** `/sessions` рендерит `Workspace`; обе вкладки исполнительных чатов смонтированы одновременно (dual-mount `ChatPage`), переключение между вкладками не прерывает текущий стрим.
- **Контролы сессии — в хедере Workspace (справа), а не в глобальном Navbar.** `WorkspaceSessionControls`: бейджи ассистента/модели, файлы сессии, тоггл Applications-pane, удаление. Они показываются только когда активна вкладка исполнителя и у него есть открытая сессия.
- **Статус-доты исполнителей — в хедере вкладок (слева от названия каждой вкладки исполнителя).** Вкладка Sessions доты не имеет. Глобальный Navbar больше не несёт индикаторов статуса Hermes/OpenCode.
- **Глубокие ссылки `/chat/:id` резолвятся в Workspace.** `ChatRouteResolver` определяет исполнителя сессии, переключает на нужную вкладку и редиректит на `/sessions`. Пользователь никогда не видит отдельной страницы чата.

### Что осталось в глобальном Navbar

После выноса контролов сессии и статус-дот в Workspace, глобальный Navbar держит только:

- **брендовое лого** (`1230[UI]`), и оно **всегда видимо** — включая мобильный режим внутри чата (раньше на мобильном в чате лого скрывалось);
- **поиск**;
- **меню пользователя** (тема, выход и т.д.).

### Тонкости для других проектов

Если повторяете этот паттерн: вынос контролов из глобального navbar в локальный хедер поверхности работает только если у поверхности есть своё стабильное состояние (Zustand-стор: активная вкладка + активная сессия на исполнитель). Dual-mount чатов безопасен только пока стримы живут в store'е на уровне app-root, а не в `useState` страницы — иначе размаунт вкладки убьёт фоновый стрим.

## 14. Мобильная адаптация

### Брейкпоинты

- **`lg` (1024px)** — главный порог: до него mobile, после desktop.
- **Один порог** лучше, чем три. Это упрощает принятие решений: либо mobile-layout (вертикальный стек, bottom-nav, full-width), либо desktop-layout (сайдбар, двухколоночный).

### Навигация

- **На десктопе** — горизонтальный navbar сверху (бренд + поиск + меню пользователя; статус-доты и контролы сессии вынесены в Workspace — см. §13).
- **На мобильном** — fixed bottom-nav (50px высота, учитывает `env(safe-area-inset-bottom)`).
- **Брендовое лого всегда видимо**, включая мобильный режим внутри чата.
- **Активный таб** на mobile показывает иконку + label, неактивные — только иконку (экономит место).
- **`aria-label="Primary"`** на обоих вариантах навигации.

### Фиксированные нижние панели

- **Mobile nav** — 50px.
- **Chat input** (если есть) — над mobile nav, итого `50px + высота инпута`.
- **Main content** получает `padding-bottom: calc(...)` с учётом safe-area.
- **Z-index:** mobile nav = 50, chat input = 55 (всё, что выше 50 — над mobile nav).

### Тач-таргеты

- Минимум **44×44 px** на любую интерактивную зону (кнопки, табы, чекбоксы).
- На мобильной навигации — `h-9 w-9` для иконок внутри `flex-1` контейнера, даёт комфортный тап.

## 15. Хранение визуальных активов

| Тип | Где хранить | Почему |
|-----|-------------|--------|
| Шрифты (брендовые, моно) | `public/fonts/*.woff2` | Без хеша в URL — кешируются; `font-display: swap`; нет зависимости от CDN |
| Шрифты (стандартные) | Google Fonts `<link>` в `index.html` с `preconnect` | Минимальный вес основного бандла |
| SVG-иконки (набор) | `public/icons.svg` (sprite) или `lucide-react` | Один HTTP-запрос, tree-shaking через библиотеку |
| Favicon | `public/favicon.svg` | SVG масштабируется под любой DPI |
| Иллюстрации (React-компоненты) | `src/assets/illustrations.tsx` | Можно стилизовать через `currentColor` / props |
| Растровые изображения (hero, превью) | `src/assets/*.png` | Импортируются в код — Vite даёт хеш, оптимизация |
| Логотип | `public/favicon.svg` + inline `<span>` в `Navbar` | SVG + текст; легко менять цвет через `currentColor` |

### Принципы

1. **Не класть растровые иконки** — только SVG (через `lucide-react` или sprite).
2. **Не использовать эмодзи** в интерфейсе — только иконки из набора.
3. **`public/`** — для файлов, на которые нужны прямые URL (favicon, шрифты, статические манифесты).
4. **`src/assets/`** — для модулей, импортируемых в JS/TS (даёт хеш в имени).
5. **Анимированные декоративные эффекты** (как `brand-bracket` с плавной сменой цвета) — через CSS-транзишены, не JS-анимации.

## 16. Производительность

- **Lazy-импорт страниц** через `React.lazy()` + `Suspense fallback={<PageSkeleton />}`.
- **Vite proxy** для `/api` в dev — нет CORS-проблем.
- **Long lists** — `react-virtuoso` вместо ручной пагинации.
- **Шрифты** — `preconnect` + `font-display: swap` + `woff2`.
- **Tree-shaking** — иконки и утилиты импортируются по именам, а не через общий barrel-файл.
- **CSS** — Tailwind автоматически удаляет неиспользуемые классы; кастомные стили — в одном `index.css`.

## 17. Доступность (a11y)

- **Семантика:** `nav`, `main`, `aside`, `header`, `footer` для лейаутов; `role="dialog"`/`"status"` где нативной семантики не хватает.
- **Фокус:** visible outline + focus-trap в модалках + восстановление фокуса при закрытии.
- **`aria-label`** на иконочных кнопках без текста (из i18n: `t('toast.dismiss')`, `t('nav.toggleDarkMode')`).
- **Tab order** = визуальный порядок (не переопределять `tabindex` положительными числами).
- **`prefers-reduced-motion`** отключает анимации (см. `@media (prefers-reduced-motion: reduce)`).
- **Print-стили** скрывают навигацию и формы, приводят цвета к светлой теме.
